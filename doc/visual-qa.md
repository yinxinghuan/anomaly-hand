## 重大重做与判断

### P1 — 游戏界面精致度低于 C 风格测试卡

- 位置：原选人页、战斗页、奖励页。
- 观察：早期版本依赖大量简单边框、纯色面板和程序化几何敌人；角色卡本身较精美，但游戏外壳像功能原型。
- 影响：玩家无法从界面感受到收藏卡世界，角色测试图与实际游戏形成明显品质落差。
- 修复：
  - 选人页改为一张大幅收藏档案卡 + 横向 8 人索引。
  - 3 名敌人改为独立 C 风格高细节 raster 档案图。
  - 战斗桌、战术牌、奖励和规则面板共享无人物丝网印刷位图底板。
  - 敌我档案改为前后压叠构图，手牌使用位图裁切、专色遮罩和纸张边缘。
- 复验：最终选人、战斗、签名技、奖励和结果页均像同一套地下异能档案，而不是“精美角色 + 通用 HUD”。

### P1 — 战后强化缺少选择欲与收藏感

- 位置：战后强化页。
- 观察：旧版虽然沿用了战斗底板，但强化仍像三条信息选项，图形密度、物件感与角色测试卡存在落差。
- 影响：两场战斗之间的奖励峰值偏弱，无法形成“获得稀有改造”的期待。
- 修复：重构为三张纵向改造卡；每张拥有独立语义专色、几何徽记、类型、大数值、名称和效果说明。选中后保留高饱和边框与“改造已装订”章，其余两张降噪，确认状态停留 620 毫秒后进入下一场。
- 复验：390 × 844 和 320 × 568 均完整显示三张卡；点击状态无裁切、无瞬时熄灭，三列宽度一致。8 位角色逐一在两种尺寸进入强化页，人物脸部均位于卡面上方可视区，双眼完整可见且不压标题；共 16 个状态控制台零错误。

### P1 — 中英文与数字缺少专属字体气质

- 位置：标题、敌人名、卡名、生命、伤害、档案编号与结果数值。
- 观察：旧版正文与标题主要依赖 Onest、系统中文字体和 JetBrains Mono；信息清楚，但英文、中文和数字像通用产品 UI，没有把“地下档案印刷物”的气质带进排版。
- 影响：角色与位图已经精致，文字仍会让界面回落到普通网页感，阿拉伯数字尤其缺少识别度。
- 修复：接入 Noto Sans SC 中文子集与 Ostrich Sans Heavy 英文/数字子集；标题按语言分别控制字距；主要数字启用等宽排列、轻微斜切与 1 px 红青套印，小型协议标签继续使用 JetBrains Mono，正文继续使用 Onest/CJK 回退。
- 复验：字体加载检查通过；中英文 390 × 844、英文 320 × 568 均无横向溢出。最终新字体版本自动完成三场战斗，第一局 20 回合获胜，控制台零错误。

### P1 — 生图卡纸出现错误视觉内容

- 位置：制作期战术卡面。
- 观察：第一轮被角色参考污染，出现熊与人物；第二轮出现手机外框、紫色霓虹、字母和无关人物。
- 影响：破坏 C 风格并造成卡牌语义混乱。
- 处理：两轮全部拒绝，不接入源码。最终从合格的无人物战斗底板中裁切不同区域，用 CSS 语义专色区分突破、守备、技术和签名牌。
- 复验：`dist/` 只包含 3 张敌人位图和 1 张战斗底板，不包含被拒绝卡面。

### P1 — 初始平衡无法稳定通关

- 位置：完整三场流程。
- 观察：策略自动玩家连续 4 局都到达第三场，但全部失败。
- 影响：不符合信息流 3–5 分钟短局和首次玩家合理胜率。
- 修复：敌人生命从 `20/28/40` 调整为 `18/24/34`，攻击从 `5/7/9` 调整为 `4/6/7`，场间恢复从 4 提升至 6。
- 复验：调整后连续两次完整测试都第一局通关，分别为 27 和 28 回合，剩余生命 15 与 17。

## 最终观察

- 选人页第一眼是一张完整收藏卡；被动与签名技都在大卡内阅读，8 人索引只承担切换。
- 敌方位图与英雄位图拥有一致的红/青专色、网点、磨损和档案边缘，资产不再显得贴上去。
- 390px 战斗页保留充分人物尺度；320px 下敌我重叠仍可辨认，三张手牌各为 104 × 150。
- 战术牌底纹丰富，但下方暗色阅读带保证中英文说明可读。
- 中文标题、英文展示字和仪表数字现在具有明确分工；`01/08`、生命、伤害和结算数字在缩略画面下也能形成稳定识别锚点。
- 奖励页使用人物水印、共享战斗底板和三张独立改造卡；语义徽记、数值与装订章让奖励阶段成为明确的视觉峰值。
- 胜利与失败使用同一英雄档案，仅通过青蓝封存章/朱红断档章区分，不依赖颜色单独传达结果。
- 严格 UI audit 无功能 Emoji；所有图标来自同一套内联 SVG。

## 战斗手感重做复验（2026-07-18）

- 新增 8 位行动员的透明战损 WebP。每张均保持睁眼、身份锚点、同一丝网印刷媒介，并在未格挡受伤后由 `playerState` 切换；恢复或进入下一遭遇后回到战备状态。
- 新增出牌四段状态：`commit → impact → discard → deal`。牌在约 210 ms 命中舞台，三张后续手牌按 70 ms 间隔发入；签名牌延长峰值并提高音画强度。
- 新增大号 `S/A/B/C` 档案评语、浮动数值、战术分与精准连段；评级文本同时承担“刚才效果”和“下一步战术是否正确”的说明，避免只靠晃动或颜色表达。
- 构建、严格 UI 审计、相对资源路径审计通过；生产包包含 8 张战损素材，且没有功能 Emoji 或根绝对资源路径。

### P1 — 回合归属与胜负去向没有被明确说明

- 观察：战斗内的评分评语能够解释一张牌的效果，却不能说明当前轮到谁、敌人即将做什么，或击破后为何突然切进强化/结果。
- 修复：新增全屏档案章节牌。它覆盖“遭遇开始 → 你的回合 → 敌方回应 → 你的回合 → 强化 / 胜利 / 失败”全部关键切换；敌方章节同时显示公开意图和数值，胜负章节先交代原因再进入结果。
- 验收：章节牌在输入锁定期间出现，收束后再开放牌；文字、颜色、章节标签三重表达，不让用户仅凭场景切换猜测状态。

### P0 — 敌方回合没有被感知为真实威胁

- 观察：此前敌方标题虽会出现，但实际结算被放在标题之后较晚的位置；防护和蓄力没有独立落点，玩家容易把连续发牌误解为始终由自己行动。
- 修复：新增常驻“行动控制权”HUD，在敌方阶段切为红色“敌方回合”；敌方标题、战场内 `HOSTILE EXECUTION` 警示、敌人档案前冲和意图闪烁同步出现。敌方攻击、防护、蓄力均在敌方阶段内执行并输出独立评语；攻击把生命从 30 降到 26 的可测结果，随后才显示下一张公开意图并交还卡牌。
- 复验：本地真实浏览器用 Smith black 的首回合“突破”测试：玩家造成 6 点（敌人 18 → 12）后，HUD 从“行动员回合”切为“敌方回合”，章节显示“敌方回应 / 攻击 4”，随后行动员生命从 30 → 26，下一意图变为“防护 3”，最后 HUD 回到“行动员回合”且新三张手牌可操作。

### P0 — 敌方动作被章节牌遮住，规则入口无法打开

- 观察：首轮复验发现敌方前冲与执行警示在章节牌仍遮盖战场时已接近结束；伤害虽然发生，视觉动作没有在无遮挡的舞台上持续。另一个真实浏览器点击发现规则按钮只进入按下态，未挂出对话框。
- 修复：将敌方时间轴改为“250 ms 敌方章节 → 760 ms 敌方现身 → 920 ms 效果落点 → 520 ms 后发牌”，并把可见执行文案由 CSS 伪元素改为 React 状态节点；HUD 增加“手牌展开”中间态，只有牌可点时才显示“行动员回合”。菜单、规则、静音和规则关闭统一使用 `onClick`，战斗出牌仍使用 `onPointerDown`。
- 复验：真实浏览器快照在敌方阶段同时显示“敌方回合 / 敌方执行中 / 受到 4 点伤害 / 30 → 26”；蓄力后下一意图显示“攻击 9”和“蓄力锁定 +5”；治疗后人物回到非战损图且生命为 29/30。规则对话框可打开，包含四条规则与“关闭规则”按钮，关闭后 DOM 中不再存在对话框。
