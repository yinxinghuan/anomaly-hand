## 当前定位

这是一个以 Aigram 玩家形象为行动员的竖屏无尽卡牌战斗游戏。玩家每回合观察敌方公开意图，从 3 张战术牌中选择 1 张；敌方完成行动后，双方进入下一回合。当前人物池里未被选择的全部行动员会依次成为敌人，清空后进入生命与攻击更高的新循环，直到玩家生命归零并提交最高战术分。

## 已完成的正式体验

- 随机 5 人的单行行动员选择、可停留的“活体档案”机制说明、敌我依次入场的遭遇导入，以及明确的玩家/敌方回合交接。
- 8 位行动员、6 张基础战术牌、8 种签名技、5 种强化与无限敌人循环；敌方每一回合都执行公开的攻击、防护或蓄力意图。
- 两拍战斗结算：先说明发生的效果，再展示对象与明确数值；评级、分数、纸屑、套印射线、电弧、声音和受击立绘共同提供即时反馈。
- 首发英雄、战术牌和强化均使用完整 WebP 卡面；人物、场景与方向边界在单图中完成，不使用透明扣底人物去拼统一底板。
- 生命已升级为敌我各自的完整度仪表：身份标签、30 px 当前生命、最大值、11 px 断裂进度条、可选格挡读数与低血量危险描边均常驻可读。
- 中文/英文界面、专属数字字形、Web Audio 合成音效、移动端触控与键盘 `1/2/3` 输入均已接入。
- 用户首次“接入档案”时后台生成一张完整个人行动员卡；击败敌人会永久封存记录，4/8/12…张时排队生成白名单内的异变指令。个人生成不会阻塞战斗，前台等待最多 12 秒。
- 平台排行榜以独立永久 UUID 分区；仅在死亡结算时提交一次分数，榜单中的他人资料可直接打开。

## 已锁定的产品与视觉原则

- 核心节奏是“公开敌方意图 → 打一张牌 → 敌方回应 → 新手牌”，不是永远由玩家连续操作。
- 所有环节必须交代下一步：遭遇识别、敌我进场、玩家回合、敌方回应、强化、轮次封存与失败都使用可读的标题页或阶段提示。
- A「漆金战谱」、B「星骸圣典」、C「异能档案」是当前可随机生成个人卡的已验收方向；D/E/F 仍是探索定义，未进入线上随机池。
- 角色面部和眼睛优先于任何装饰层；战损由完整卡面上的裂痕、暗角、血痕与表情变化表达。
- 战斗视觉保持“地下异能机构丝网印刷档案”气质：炭黑、骨白、朱红、酸性青蓝为主；不得回退为通用赛博玻璃面板、厚重奇幻卡框或 Emoji 图标界面。
- 生命、意图、手牌和结算是战斗注意路径。生命不能再退化为角色卡边缘的微型文字，也不能只通过颜色表达敌我或增减。

## 后续接手注意

- 改玩法先更新 `doc/requirements.md`；改界面或动效先更新 `doc/visual.md`；代码完成后同步 `doc/technical.md` 与 `doc/visual-qa.md`。
- 任何视觉改动必须在真实 390 × 844 运行画面复验，并检查 320 × 568 不溢出。不要只依赖静态设计稿或代码判断。
- 运行时头像生图只能走 Aigram transit；不要引入 ComfyUI、本地工作流或把用户头像保存为二进制。
- 个人档案的 `generating` 状态没有远端可恢复任务 ID：重新接入时应沿用已锁定风格重新请求，不能永久停在“正在建立档案”。
- 上线前必须运行 `npm run build`、UUID 校验、相对资源路径审计，并在推送后检查线上 bundle，而不是只看 `git push` 或工作流成功。
